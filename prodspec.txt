# 🗄️ 3. DATABASE SCHEMA (PRODUCTION-READY)

Below is a clean, normalized schema your dev can implement directly.

---

## 3.1 laws

```sql
CREATE TABLE laws (
  id UUID PRIMARY KEY,
  slug TEXT UNIQUE,
  title TEXT NOT NULL,
  short_title TEXT,
  jurisdiction_id UUID,
  issuing_body TEXT,
  status TEXT,
  content_type TEXT,
  summary_short TEXT,
  summary_long TEXT,
  official_url TEXT,
  adopted_date DATE,
  effective_date DATE,
  last_reviewed_at TIMESTAMP,
  is_published BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

---

## 3.2 obligations

```sql
CREATE TABLE obligations (
  id UUID PRIMARY KEY,
  law_id UUID REFERENCES laws(id),
  title TEXT,
  description TEXT,
  category TEXT,
  priority TEXT,
  severity TEXT,
  citation TEXT,
  action_required TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 3.3 applicability_rules

```sql
CREATE TABLE applicability_rules (
  id UUID PRIMARY KEY,
  law_id UUID REFERENCES laws(id),
  name TEXT,
  rule_json JSONB,
  weight FLOAT DEFAULT 1,
  enabled BOOLEAN DEFAULT true
);
```

---

## 3.4 assessments

```sql
CREATE TABLE assessments (
  id UUID PRIMARY KEY,
  user_id UUID,
  company_profile JSONB,
  product_profile JSONB,
  technical_profile JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 3.5 assessment_results

```sql
CREATE TABLE assessment_results (
  id UUID PRIMARY KEY,
  assessment_id UUID REFERENCES assessments(id),
  law_id UUID REFERENCES laws(id),
  relevance_score FLOAT,
  applicability_status TEXT,
  rationale TEXT,
  triggered_conditions JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 3.6 checklists

```sql
CREATE TABLE checklists (
  id UUID PRIMARY KEY,
  assessment_id UUID,
  title TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 3.7 checklist_items

```sql
CREATE TABLE checklist_items (
  id UUID PRIMARY KEY,
  checklist_id UUID,
  obligation_id UUID,
  title TEXT,
  description TEXT,
  status TEXT DEFAULT 'not_started',
  priority TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

# 🧠 5. RULES ENGINE SPEC

## 5.1 Core Principle

Deterministic > AI

---

## 5.2 Rule Structure

```json
{
  "all": [
    {
      "fact": "target_geographies",
      "operator": "contains",
      "value": "EU"
    },
    {
      "fact": "use_case",
      "operator": "in",
      "value": ["hr", "recruitment"]
    }
  ]
}
```

---

## 5.3 Engine Logic

Pseudo-code:

```
for each law:
  for each rule:
    evaluate(rule, input)
    if match:
      score += weight

final_score = normalize(score)
status = map_to_status(final_score)
```

---

## 5.4 Status Mapping

| Score    | Status         |
| -------- | -------------- |
| >0.75    | likely_applies |
| 0.4–0.75 | may_apply      |
| <0.4     | unlikely       |
